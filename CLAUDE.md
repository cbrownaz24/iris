## Project Overview

IRIS (Imagination with auto-Regression over an Inner Speech) is a model-based reinforcement learning agent that learns entirely inside an imagined world model. The world model combines a VQ-VAE discrete autoencoder (image <-> tokens) with an autoregressive Transformer that predicts next-state tokens, rewards, and episode terminations. The agent is evaluated on the Atari 100k benchmark.

## Architecture

### Three-Component Agent (`src/agent.py` -> `Agent`)

The `Agent` bundles three independently-trained components, each with its own optimizer and training schedule (staggered start epochs: tokenizer@5, world_model@25, actor_critic@50):

1. **Tokenizer** (`src/models/tokenizer/`) — VQ-VAE that converts 64x64 RGB frames into K=16 discrete tokens from a vocabulary of N=512. Encoder CNN -> codebook quantization -> decoder CNN.

2. **World Model** (`src/models/world_model.py`) — Takes interleaved sequences of observation tokens and action tokens. Each "block" = 16 obs tokens + 1 action token = 17 tokens. Up to 20 blocks (340 tokens max). A GPT-style Transformer predicts next obs tokens, rewards {-1,0,1}, and episode ends {0,1} via three separate heads.

3. **Actor-Critic** (`src/models/actor_critic.py`) — CNN + LSTM policy that operates on reconstructed 64x64 frames. Trained purely on imagined trajectories generated by rolling out the world model.

### Token Sequence Layout (World Model)

Within each block of `tokens_per_block=17`:
- Positions 0-15: observation tokens (from VQ-VAE encoder)
- Position 16: action token

The `Embedder` and `Head` classes in `src/models/slicer.py` use block masks to route tokens to the correct embedding tables and prediction heads.

### Imagination Pipeline

`WorldModelEnv` (`src/envs/world_model_env.py`) wraps the world model for step-by-step generation:
- Maintains a KV cache (`src/models/kv_caching.py`) for efficient autoregressive generation
- Each `step()` feeds action token, reads reward/done from the action-position head, then autoregressively samples K new observation tokens
- Refreshes KV cache when approaching `max_tokens` limit

### Training Loop (`src/trainer.py`)

Each epoch: collect real experience -> train tokenizer -> train world model -> train actor-critic (in imagination). All three components expose a `compute_loss(batch, ...)` method returning `LossWithIntermediateLosses`.

### Configuration

Hydra-based config in `config/`. Main file: `config/trainer.yaml`. Sub-configs:
- `config/world_model/default.yaml` — TransformerConfig (layers, heads, embed_dim, etc.)
- `config/tokenizer/default.yaml` — VQ-VAE vocab size, encoder/decoder arch
- `config/env/default.yaml` — Atari environment settings (game ID set at launch)
- `config/actor_critic/default.yaml`, `config/datasets/default.yaml`

Config classes are instantiated via `hydra.utils.instantiate()`. The world model config maps to `TransformerConfig` dataclass in `src/models/transformer.py`.

### Run Output Structure

Each run outputs to `outputs/YYYY-MM-DD/hh-mm-ss/` containing checkpoints, config snapshot, media (episodes/reconstructions), and a copy of src/scripts.

## Key Conventions

- The world model's `forward()` accepts `(tokens, past_keys_values)` and returns `WorldModelOutput` with `output_sequence`, `logits_observations`, `logits_rewards`, `logits_ends`.
- `init_weights` in `src/utils.py` applies N(0, 0.02) init to Linear/Embedding layers and unit init to LayerNorm.
- `configure_optimizer` in `src/utils.py` sets up AdamW with weight decay excluded for LayerNorm and Embedding parameters.
- Rewards are clipped to {-1, 0, 1} for the world model loss (3-class classification).
